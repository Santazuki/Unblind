# Unblind 项目定位与创新突破

## 一、核心定位

Unblind 是一个为 **Claude Code 中纯文本模型（如 DeepSeek）** 提供视觉理解能力的轻量级 Skill。它不追求成为“万能视觉接入层”，也不强制引入 MCP 的复杂度和资源开销，而是专注于一件事：**以最低的 Token 成本、最快的响应速度，让不具备原生视觉能力的模型能够“看见”并理解图片**。

在 DeepSeek 和 Mimo 双双宣布永久折扣的背景下，Unblind 的在线视觉方案拥有了长期可持续的性价比优势，成为日常开发中随手可用的工具。

## 二、与现有方案的差异化

| 维度       | Unblind                            | 典型 MCP 方案（如 opencode-vision） | 典型全能 Skill（如 vision-support） |
| :--------- | :--------------------------------- | :---------------------------------- | :---------------------------------- |
| 形态       | Skill（指令集）                    | MCP Server（独立服务）              | Skill（大型指令集）                 |
| 上下文占用 | 极低（三级加载，初始 <200 tokens） | 高（全量加载 ~18k tokens）          | 中等偏高                            |
| 依赖       | 零依赖（纯 Node.js）               | 常需 Docker / Python                | 中等                                |
| 启动速度   | 毫秒级                             | 秒级                                | 毫秒级                              |
| 适用场景   | 快速看图、低成本分析、集成到对话流 | 企业级、多模型路由、复杂工作流      | 功能全面的视觉工具包                |
| 核心哲学   | 做减法：够用即可                   | 做加法：标准化、可复用              | 做乘法：一站式解决方案              |

**结论**：Unblind 填补了「轻量、即用、低成本视觉增强」的空白，是与 MCP 和全能 Skill 互补而非竞争的存在。

## 三、创新突破（可逐步实现）

以下创新点尚未被现有方案充分实现，可作为 Unblind 的差异化亮点：

### 1. 混合视觉流水线（Hybrid Pipeline）

- **问题**：纯云端调用有延迟和成本；纯本地模型准确率有限。
- **方案**：先用本地轻量模型（如预置的 MobileNet 或 TensorFlow Lite 分类器）做预分析，提取基础特征（有无文字、模糊度、大致类别）。若置信度超过阈值（如 0.85），直接返回结果，**零成本**；否则才调用云端模型。
- **收益**：显著降低 API 调用次数，同时保证复杂场景的准确率。

### 2. 上下文感知的图片分块

- **问题**：大图（长截图、高分辨率图表）直接发送会消耗大量 token，且模型可能忽略关键区域。
- **方案**：将图片按网格切分，根据用户问题中的关键词（如“左上角按钮”、“第三行数据”）优先分析相关区块，而非全图发送。
- **收益**：Token 消耗降低 50%~80%，细节识别准确率提升。

### 3. 视觉链式思考（Visual Chain-of-Thought）

- **问题**：现有方案只返回最终描述，无法解释模型“看到了什么”。
- **方案**：强制模型按「区域定位 → 特征提取 → 关系推理 → 综合结论」的分步格式输出结果，并在 Skill 层面提供基于坐标的简易注意力热力图（用字符画标注）。
- **收益**：增强可解释性，便于调试和二次处理。

### 4. 可编程结构化输出

- **问题**：自然语言描述难以被脚本或下游工具直接使用。
- **方案**：允许用户在请求中指定输出格式（JSON / YAML / CSV）。Skill 在调用视觉模型后，再用本地规则或轻量 LLM 将描述转换为目标结构。
- **收益**：视觉分析结果可无缝接入数据管道和自动化流程。

### 5. 视觉暂存区（Visual Scratchpad）

- **问题**：用户在同一会话或多轮对话中反复发送相似图片，每次都重复调用 API。
- **方案**：基于图片感知哈希（pHash）进行缓存，相似图片（汉明距离小于阈值）直接返回缓存结果；支持按 TTL 或手动清除缓存。
- **收益**：重复图片的分析成本降至接近零。

## 四、总结

Unblind 的独特价值不在于「做了视觉能力」，而在于 **「用最轻量、最透明、最具创新性的方式实现了视觉能力」**。它不需要与 MCP 或全能 Skill 正面对抗，而是服务于那些追求低成本、快速集成、对话上下文友好的开发者场景。通过上述创新点的逐步落地，Unblind 完全有能力从一个个人工具成长为社区中具有鲜明特色的 Skill 标杆。
